ShaderProgram Class

Extends RenderingContextUser
Defined in: src/davinci-eight/programs/ShaderProgram.ts:3
Module: EIGHT

The role of a ShaderProgram is to manage the WebGLProgram consisting of a vertex shader and fragment shader. The ShaderProgram must be able to provide introspection information that describes the program.

contextFree

()

Inherited from RenderingContextUser: src/davinci-eight/core/RenderingContextUser.ts:9

Called to request the dependent to free any WebGL resources acquired and owned. The dependent may assume that its cached context is still valid in order to properly dispose of any cached resources. In the case of shared objects, this method may be called multiple times for what is logically the same context. In such cases the dependent must be idempotent and respond only to the first request.

Called to inform the dependent of a new WebGLRenderingContext. The contextId parameter allows the implementation to ignore the notification if it has already received the specified identifier.

Parameters:

    The WebGL rendering context.

  • contextId String

    The identifier of the context gain event.

contextLoss

()

Inherited from RenderingContextUser: src/davinci-eight/core/RenderingContextUser.ts:27

Called to inform the dependent of a loss of WebGLRenderingContext. The dependent must assume that any cached context is invalid. The dependent must not try to use and cached context to free resources. The dependent should reset its state to that for which there is no context.
